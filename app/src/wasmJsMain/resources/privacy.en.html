3. Server Operation, Technical Connection Data and IP Addresses

I do not conduct independent IP tracking and do not analyze IP addresses for marketing purposes. At the application level, the service is configured so that IP addresses and technical connection data are not deliberately stored permanently.

Independently of this, the hosting provider (Hetzner) inevitably processes technical connection data, in particular IP addresses, as part of the technical operation and security of the infrastructure, to the extent necessary for provision, operation, error analysis and protection against attacks. Which specific technical log data Hetzner stores in individual cases and for how long depends on the systems used and the requirements of the hosting provider. This data is generally not provided to me.

Categories of processed data: Technical connection data (IP address, timestamp, HTTP request data, browser information).

Legal basis: Art. 6 para. 1 lit. f GDPR (legitimate interest in secure and reliable server operation and protection against misuse).

Obligation to provide data: The processing of technical connection data is mandatory for accessing the website. Without this data processing, the service cannot be provided.

Third-country transfer: No transfer of personal data to third countries outside the EU or EEA takes place in the context of hosting.

For more information about data processing by Hetzner, please refer to Hetzner's privacy policy: https://www.hetzner.com/legal/privacy-policy/

4. Steam Login (Optional) and Third-Party Services

The ONI Seed Browser can generally be used without registration. Optionally, you can log in via Steam Login.

For login, an external authentication service from Steam (Valve Corporation, USA) is used. Communication takes place between your device and Steam. I only receive your SteamID. No other Steam profile information (e.g., username, avatar, or friends list) is retrieved from Steam.

Categories of processed data: Online identifiers (SteamID, cryptographic hash of the SteamID).

Recipients: During login, data is transmitted to Steam or Valve Corporation (USA). I subsequently process the SteamID received from Steam to perform the login and for the functions described below.

Third-country transfer: As part of Steam authentication, data is transferred to Valve Corporation, USA. Valve has committed to complying with EU Standard Contractual Clauses. For more information, please refer to Steam's privacy policy: https://store.steampowered.com/privacy_agreement/

For technical authentication, a JWT (JSON Web Token) is generated, which contains the SteamID and a cryptographic hash of this ID.

To remain logged in, this JWT can be stored locally in the browser (see section "Storage of Settings and Login Status (localStorage)").

Purpose: Authentication, abuse prevention, and assignment of contributions and requests.

Legal basis: Art. 6 para. 1 lit. b GDPR (provision of the optional login function) and Art. 6 para. 1 lit. f GDPR (abuse prevention).

Obligation to provide data: Login via Steam is voluntary. Without login, you can use the service in read-only mode, but cannot upload Seeds/Maps or submit requests.

5. Seeds/Maps, Mod Uploads, Requests and Leaderboard

Users can submit Seeds/Maps to the service's API via a mod for the game Oxygen Not Included. A valid Steam auth token (JWT) is required for this.

Additionally, logged-in users can request Seeds/Maps to be added to the database (requests), and optionally set a name for a leaderboard.

The following information is processed in the database:

  • the submitted Seed or Map data structure (contribution data)
  • an assignment to the SteamID of the contributing user (for abuse prevention and traceability)
  • if voluntarily provided: a name set by the user for the leaderboard

Categories of processed data: Online identifiers (SteamID, cryptographic hash), game data (Seeds/Maps), optional user inputs (leaderboard name).

Leaderboard: The leaderboard shows which user has contributed how many Seeds/Maps. A public association with a person is only possible if a user voluntarily provides a name for the leaderboard.

Public delivery: Seeds/Maps are delivered publicly. Not the SteamID itself is used, but exclusively a hash of the SteamID, so that no direct assignment to a person is possible.

Retention period:

  • For requested Seeds/Maps (requests), the SteamID is deleted from the database once the requested Seed or Map has been successfully created or added.
  • For Seeds/Maps submitted via the mod, the SteamID remains stored together with the respective contribution for the duration of its existence. No automatic deletion takes place. Upon request of the contributor, a specific contribution (Seed/Map) can be deleted. The deletion removes the assignment of this contribution to the SteamID.

Legal basis: Art. 6 para. 1 lit. b GDPR (provision of functions when you log in) and Art. 6 para. 1 lit. f GDPR (abuse prevention, system integrity, traceability of contributions).

6. Storage of Settings and Login Status (localStorage)

The ONI Seed Browser stores certain data locally in the browser (localStorage). This storage takes place exclusively on your device.

a) Search filters and markers
Users can save search filters and mark coordinates. This information is stored exclusively locally and is not transmitted to the server.

b) Login status (JWT)
When you log in via Steam, the generated authentication token (JWT) is stored locally in the browser so that you remain logged in and can use the corresponding functions. The JWT is not automatically time-limited. A dedicated logout function is not implemented. To end your login session, delete the website data in your browser (clear localStorage).

The JWT is transmitted to my server during authenticated API requests to assign your SteamID and provide the desired functions.

Retention period:
The data stored in localStorage remains there until you delete it yourself in the browser.

Legal basis (device storage):
§ 25 para. 2 no. 2 TTDSG (technically required storage/retrieval for providing a function requested by the user).

Legal basis (data processing):
Art. 6 para. 1 lit. b GDPR and Art. 6 para. 1 lit. f GDPR.

7. Encryption

All data transmission is encrypted via HTTPS (TLS).
